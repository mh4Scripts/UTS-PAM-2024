Anda dapat mengakses Prototype dari Sini : LINK PROTOTIPE FIGMA

  • Splash Screen *harus ada

    1. Logo bisa apa saja
    2. Beri penundaan tiga detik (3000ms) untuk langsung pindah ke halaman berikutnya
    3. Periksa apakah Anda sudah masuk atau belum menggunakan DataStore Preference
      Jika belum, Anda akan diarahkan ke halaman Login atau ke halaman Onboarding Motion Layout (bonus)
      Jika sudah, Anda akan diarahkan ke halaman Beranda
    4. Panduan Cara Membuat Splash Screen

  • Halaman Login *harus ada

    1. Buat halaman Login
    2. Pengguna dapat masuk
    3. Pengguna dapat klik tombol registrasi untuk pindah ke halaman registrasi
    4. Periksa apakah nama pengguna dan kata sandi yang dimasukkan tersimpan di DataStore / Firebase Auth Tutorial Firebase
    5. Jika berhasil masuk, ia akan diarahkan ke halaman dasbor dan status "masuk" akan disimpan di DataStore Preference
    6. Buat email: admin dan password: PAM2024 sebagai akun tes
    7. Logout akan kembali ke halaman login

  • Halaman Registrasi *harus ada

    1. Buat halaman Registrasi untuk mendaftar akun (bisa dalam halaman yang sama dengan halaman Login atau buat halaman baru)
    2. Setidaknya terdapat nama pengguna aplikasi, kata sandi, nama pengguna GitHub, nik, dan email
    3. Data disimpan di DataStore / Firebase Auth

  • Halaman Beranda *harus ada

    1. Buat Halaman Beranda (bisa Activity atau Fragment)
    2. Tampilkan daftar pengguna dari API reqres.in seperti Contoh Kode di materi sesi 5 menggunakan recyclerview (tampilkan foto, nama pengguna, dan email) menggunakan Retrofit
    3. Dapat dicari menggunakan searchview seperti pada tugas individu sebelumnya

  • Halaman Profil *harus ada

    1. Buat Halaman Pengguna (bisa Activity atau Fragment)
    2. Tampilkan login pengguna yang minimal terdiri dari foto dalam bentuk huruf pertama dari nama pengguna Tutorial, nama pengguna, nama pengguna GitHub, dan Email
    3. Juga, sertakan fungsi logout untuk pindah ke halaman Login agar pengguna lain dapat masuk/registrasi

  • Halaman OnBoarding menggunakan Motion Layout (bonus)

    1. Buat Halaman OnBoarding menggunakan Motion Layout
    2. Buat minimal tiga papan, kontennya bisa apa saja
    3. Panduan Proyek OnBoarding
    4. Tutorial: Klik disini
    5. Contoh Kode: Klik disini

  • Halaman Detail Pengguna (bonus)

    1. Buat Halaman Detail Pengguna (bisa Activity atau Fragment)
    2. Teruskan Argumen atau Intent dari Beranda, menggunakan id untuk mendapatkan data pengguna dari API https://reqres.in/api/users/2
    3. Tampilkan data pengguna dari Respons API
